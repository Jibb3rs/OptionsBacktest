PROJECT OVERVIEW
================================================================================

This is a comprehensive Python-based options backtesting system with a modern
PyQt5 + Apache ECharts GUI that allows traders to backtest options strategies
on historical data using QuantConnect's platform. The system generates production-ready QuantConnect 
algorithm code with advanced entry filters, technical indicators, and risk 
management features.

CREATOR: Justin (experienced forex trader and developer)
BACKGROUND: 
- Built sophisticated "JustinDPC" (Day Performance Concept) EA for MetaTrader 4
- Expanding into options trading with systematic approach
- Professional experience: ByteDance, BlackRock, J.P. Morgan Chase
- Education: University of Essex (Bachelor's), Temasek Polytechnic (Diploma)
- Skills: MQL4, Python, API integration (Tiger Brokers), multiple trading platforms

PROJECT GOALS:
1. Backtest options strategies with professional-grade filters
2. Generate QuantConnect code automatically (no manual coding)
3. Analyze trade performance with MAE/MFE metrics
4. Compare multiple strategies side-by-side
5. Support 16+ technical indicators for entry filtering
6. Enable advanced filters (VIX, IV Rank, Market Profile, Greeks)
7. Provide institutional-quality backtesting for retail traders

================================================================================
STRIKE SELECTION METHODS
================================================================================

The system supports 4 methods for selecting option strikes:

1. DELTA-BASED (Recommended) ⭐
   - Automatically adjusts for volatility
   - Example: 0.16 delta = ~84% OTM probability
   - When VIX high: Strikes placed farther out
   - When VIX low: Strikes placed closer
   - Most professional approach

2. FIXED POINTS
   - Static distance from current price
   - Example: 50 points below for puts
   - Simple but doesn't account for volatility
   - Good for index options (SPX, SPY)

3. PERCENTAGE-BASED
   - Percentage of current price
   - Example: 5% OTM
   - Scales with price level
   - Good for stocks of varying prices

4. ATR-BASED
   - Uses Average True Range (volatility measure)
   - Example: 2x ATR for puts
   - Adapts to recent price movement
   - Good for volatile underlying

DELTA GUIDE FOR IRON CONDOR:
- Short Put Delta: 0.16 (84% win rate) - Standard
- Long Put Delta: 0.08 (92% win rate) - Protection
- Short Call Delta: -0.16 (84% win rate) - Standard
- Long Call Delta: -0.08 (92% win rate) - Protection

ADVANCED FILTERS (PROFESSIONAL-GRADE)
================================================================================

VOLATILITY FILTERS:
-------------------
1. VIX Filter ⭐
   - Range: Min/Max VIX levels
   - Example: 15-50 for premium selling
   - <15 = Calm, 15-25 = Normal, >25 = Fear
   - Use Cases:
     * Sell premium: 15-50 (elevated volatility)
     * Buy options: 10-20 (before spike)
     * High fear: 25-80 (crisis opportunities)

2. IV Rank Filter
   - Range: 0-100% percentile
   - Example: 50-100 for Iron Condor
   - Compares current IV to 1-year range
   - Higher IV Rank = Better premium selling

3. Volatility Regime Filter
   - Types: low, medium, high (using ATR)
   - Automatically detects market regime
   - Match strategy to regime

PRICE ACTION FILTERS:
---------------------
4. Market Profile (TPO)
   - Signals: POC, VAL, VAH
   - Lookback: 20 days (default)
   - Use: Enter at value areas
   - POC = Point of Control (most traded price)
   - VAL = Value Area Low
   - VAH = Value Area High

LIQUIDITY FILTERS:
------------------
5. Liquidity Filter ⭐ (Highly Recommended)
   - Max Bid-Ask Spread: 5% (default)
   - Min Open Interest: 100 contracts
   - Ensures realistic fills
   - Prevents backtests showing unrealistic profits

EXPIRATION FILTERS:
-------------------
6. Expiration Cycle Filter
   - Types: any, monthly, weekly, eom, quarterly
   - Monthly: 3rd Friday
   - Weekly: Every Friday
   - EOM: Last trading day of month
   - Quarterly: Mar/Jun/Sep/Dec

RISK MANAGEMENT FEATURES
================================================================================

STOP LOSS MODES:
----------------
1. Credit-Based (Default)
   - Stop at % of credit received
   - Example: 200% of credit = 2x max loss
   - Most common for credit spreads

2. Max Loss-Based
   - Stop at % of max possible loss
   - Example: 50% of max loss
   - Conservative approach

3. Equal Dollar
   - Stop loss = profit target (in dollars)
   - 1:1 risk/reward ratio
   - Symmetric risk management

4. Price Distance
   - Stop when underlying moves X%
   - Example: Stop if SPX moves 5%
   - Based on underlying price action

PROFIT TARGETS:
---------------
- Percentage of max profit
- Example: Close at 50% of max profit
- Standard: 50-75% for credit spreads
- Locks in profits early, avoids gamma risk

POSITION MANAGEMENT:
--------------------
- Max concurrent positions
- Min days between trades
- Same-day trading limits
- Position sizing (planned)

ADVANCED GREEKS INDICATORS:
---------------------------
These are second-order and third-order Greeks that measure how the first-order
Greeks (Delta, Gamma, Vega, Theta) change with various factors:

1. Gamma
   - Rate of change of Delta
   - Measures acceleration of Delta
   - Entry Signal: Enter when Gamma is low (far from ATM)
   - Risk Signal: Exit when Gamma gets too high (near ATM)
   - Use Case: Avoid high gamma risk for Iron Condors

2. Vanna (dDelta/dIV)
   - How Delta changes with volatility
   - Positive Vanna: Delta increases as IV increases
   - Entry Signal: Enter when Vanna risk is low
   - Use Case: Manage volatility exposure

3. Charm (dDelta/dTime)
   - How Delta changes over time
   - Also called "Delta Decay"
   - Entry Signal: Understand how position Delta will evolve
   - Use Case: Forecast directional risk over time

4. Vomma (dVega/dIV)
   - How Vega changes with volatility
   - Also called "Volga"
   - Entry Signal: Second-order volatility risk
   - Use Case: Manage vega exposure in volatile markets

5. Veta (dVega/dTime)
   - How Vega changes over time
   - Entry Signal: Understand how volatility exposure decays
   - Use Case: Time-based volatility management

6. Speed (dGamma/dSpot)
   - How Gamma changes with underlying price
   - Third-order Greek
   - Risk Signal: Extreme gamma risk zones
   - Use Case: Advanced risk management

7. Zomma (dGamma/dIV)
   - How Gamma changes with volatility
   - Entry Signal: Volatility impact on gamma
   - Use Case: Complex volatility scenarios

8. Color (dGamma/dTime)
   - How Gamma changes over time
   - Risk Signal: Gamma acceleration near expiration
   - Use Case: Time decay of gamma risk

TROUBLESHOOTING GUIDE
================================================================================

COMMON ISSUES:
--------------

1. "No trades executed in backtest"
   Causes:
   - Too many filters enabled (overfitting)
   - Indicators too strict (RSI < 10 rarely happens)
   - VIX range too narrow (15-16 vs 15-30)
   - Date range too short
   Solutions:
   - Disable some indicators
   - Relax thresholds (RSI < 40 instead of < 30)
   - Widen VIX range
   - Test longer period (1+ years)

2. "Backtest shows unrealistic profits"
   Causes:
   - Liquidity filter disabled
   - Trading illiquid strikes
   Solutions:
   - Enable liquidity filter (ALWAYS)
   - Increase min open interest to 100+
   - Reduce max spread to 5%

3. "Stop loss getting hit too often"
   Causes:
   - Stop loss too tight
   - Using wrong stop loss mode
   Solutions:
   - Increase stop loss % (200% → 300%)
   - Try different mode (credit → max loss)
   - Review MAE/MFE analysis to optimize

4. "Win rate too low for Iron Condor"
   Causes:
   - Wrong market regime (trending vs ranging)
   - Deltas too aggressive (0.30 instead of 0.16)
   Solutions:
   - Enable ADX filter (want <20 for IC)
   - Use safer deltas (0.10 instead of 0.16)
   - Add VIX filter (avoid panic selling)

5. "Indicator not filtering correctly"
   Causes:
   - Wrong condition selected
   - Parameter mismatch
   Solutions:
   - Review indicator configuration
   - Check condition dropdown
   - Verify period settings

CHART TECHNOLOGY:
-----------------
- Apache ECharts via QWebEngineView (NOT matplotlib)
- Embed echarts.min.js (~1MB) into gui/assets/ (from jsdelivr)
- Reusable EChartsWidget(QWidget) wrapper class:
  * Reads echarts.min.js from local file, embeds inline in HTML
  * set_option(dict) method: serializes Python dict to JSON
  * Ready-flag queue: stores pending option if called before page loads
  * Escape backticks/backslashes in JSON for JS template literal safety
  * Auto-resize: JS window.resize + Python resizeEvent trigger chart.resize()
  * Dark theme registered via echarts.registerTheme()

CHART INTERACTIVITY (Built into ECharts):
-----------------------------------------
- tooltip: {trigger: 'axis', axisPointer: {type: 'cross'}} — crosshair on hover
- legend: {type: 'scroll'} — click legend items to toggle series
- dataZoom: [{type: 'inside'}] — scroll-to-zoom, drag-to-pan
- axisPointer: {link: [{xAxisIndex: 'all'}]} — synchronized crosshair
- Multiple y-axes: yAxisIndex per series with show: False for overlays
- Dual-grid layout: grid: [{top, height}, {top, height}] with gridIndex

DATA FLOW:
----------
- Parse CSV data in Python (pandas) into lists/dicts
- Build ECharts option dict in Python with all series, axes, grid config
- Call chart_widget.set_option(option_dict) — one call renders everything
- On refresh: call set_option() with notMerge: true to fully replace
